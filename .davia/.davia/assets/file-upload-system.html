File Upload System

File upload and management workflow.

Upload Process:

  • Frontend validates file size (max 10MB)
  • Creates FormData with file
  • POST to /api/files/upload
  • Backend validates file type and size
  • File saved to disk (./uploads/)
  • File record created in database
  • File ID returned to frontend
  • File associated with entity (CV, Application, etc.)

File Storage:

  • Location: ./uploads/
  • Structure: uploads/cvs/pool/{cvId}/
  • Naming: timestamp_randomId.extension
  • Security: Token-based download URLs

Supported Formats:

  • Documents: PDF, DOC, DOCX
  • Images: JPG, PNG (for profiles)
  • Max size: 10MB per file
