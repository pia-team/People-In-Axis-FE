Testing Strategy

Testing approach for frontend and backend.

Frontend Testing:

  • Unit Tests: Vitest for component testing
  • E2E Tests: Playwright for integration testing
  • Coverage: Test coverage reports

Backend Testing:

  • Unit Tests: JUnit for service and repository tests
  • Integration Tests: @SpringBootTest for API testing
  • Repository Tests: @DataJpaTest for database tests

Test Structure:

  • Frontend: src/test/ directory
  • Backend: src/test/java/ directory
  • Test files: *.test.ts (frontend), *Test.java (backend)

Running Tests:

  • Frontend: npm run test
  • Backend: mvn test
  • Coverage: npm run test:coverage

Test Best Practices:

  • Test business logic, not implementation
  • Mock external dependencies
  • Use test databases for integration tests
  • Keep tests fast and isolated
